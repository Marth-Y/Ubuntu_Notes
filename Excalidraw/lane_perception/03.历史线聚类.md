## 1. 作用概述

`TrySetTopoLineSplitMergeAttribute` 的主要功能是**自动判断每个 topo 点（拓扑点）是“分流点”还是“合流点”**，并设置其类型属性（`KEYPOINT_FORK` 或 `KEYPOINT_MERGE`），为后续地图结构化和车道线分叉/合流场景感知做准备。

---

## 2. 主要流程

### a. topo点与车道线分组

- 遍历所有地图线（`id_line_map_`），收集每条线关联的 topo 点（`related_topo_areas`），建立 topo_id 到线id的分组映射（`topo_areas`）。

### b. 只处理关联多条线的 topo 点

- 对每个 topo_id，如果只关联了一条线，跳过（无法判断分流/合流）。

### c. 跳过已设置类型的 topo 点

- 如果 topo 点已经有类型（不是 `UNKNOWN_TYPE`），跳过。

### d. 计算 topo 点在 body 坐标系下的 x 坐标

- 将 topo 点的世界坐标转换到当前车辆 body 坐标系下，得到 `topo_x`。

### e. 收集 topo 点附近的稀疏点

- 在 `topo_x` 前后各取一段距离（如 15 米），分别收集所有关联线的稀疏点，分为 `pts_near`（topo点前）和 `pts_far`（topo点后）。

### f. 点数不足则跳过

- 如果 `pts_near` 或 `pts_far` 点数太少（如 <10），跳过该 topo 点。

### g. 二次拟合并计算残差 RMS

- 对 `pts_near` 和 `pts_far` 分别做二次多项式拟合（`y = a0 + a1*x + a2*x^2`），计算拟合残差的均方根（RMS），分别为 `rms_near` 和 `rms_far`。

### h. 线段长度不足则跳过

- 如果拟合线段长度太短（如 <8米），跳过。

### i. 动态调整判断阈值

- 根据 `topo_x` 距离远近，动态调整判断阈值 `safe_thres`（如近处 0.45 米，远处更严格）。

### j. 判断分流/合流

- **分流点（KEYPOINT_FORK）**：如果 `rms_far - rms_near > safe_thres` 且 `(rms_far - rms_near)/rms_far > 0.5`，说明 topo 点后方线变得分散，判定为分流点。
- **合流点（KEYPOINT_MERGE）**：如果 `rms_near - rms_far > safe_thres` 且 `(rms_near - rms_far)/rms_near > 0.5`，说明 topo 点前方线分散，后方变得集中，判定为合流点。

- 如果都不满足，则 topo 点类型保持未知。

---

## 3. 关键代码片段

```cpp
if (rms_far - rms_near > safe_thres && (rms_far - rms_near)/rms_far > 0.5f) {
    topo_pt_world.type = LANDMARK_TYPE::KEYPOINT_FORK; // 分流
} else if (rms_near - rms_far > safe_thres && (rms_near - rms_far)/rms_near > 0.5f) {
    topo_pt_world.type = LANDMARK_TYPE::KEYPOINT_MERGE; // 合流
}
```

---

## 4. 物理意义

- **RMS残差**反映了点集对二次曲线的聚集度，RMS大说明点分散，RMS小说明点集中。
- 分流点：通过 topo 点后，线变得分散，RMS变大。
- 合流点：通过 topo 点后，线变得集中，RMS变小。

---

## 5. 总结

- 该函数自动分析 topo 点附近的车道线分布，通过拟合残差变化判断分流/合流属性。
- 结果用于地图结构化、分叉/合流场景识别、后续路径规划等模块。
- 关键是**二次拟合+RMS残差对比+动态阈值**，实现了物理意义上的分流/合流自动判别。
